# Security Policy

## Supported Versions

Skaal is currently supported for security fixes on the `0.4` release line only.

| Version | Supported          |
| ------- | ------------------ |
| 0.4.x   | :white_check_mark: |
| 0.3.x   | :x:                |
| < 0.3   | :x:                |

## Reporting a Vulnerability

Please do **not** report security vulnerabilities in public GitHub issues.

Instead, report suspected vulnerabilities privately through GitHub Private Vulnerability Reporting:

https://github.com/Elouen-ginat/Skaal/security/advisories/new

Please include, where possible:

- The affected version or commit
- The deployment target involved, such as `local`, `aws`, or `gcp`
- Clear reproduction steps
- A proof of concept or minimal example
- The expected impact
- Any suggested mitigation or patch

What to expect after reporting:

- An acknowledgment within 5 business days
- Follow-up updates at least every 10 business days while the report is under review
- Coordination on disclosure timing if the issue is confirmed
- A clear explanation if the report is declined, such as unsupported version, non-reproducible behavior, or non-security impact

Please avoid public disclosure until a fix or mitigation is available.

For non-sensitive bugs and general defects, use the normal public issue tracker.
